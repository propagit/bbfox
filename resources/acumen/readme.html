Documentation conventions

Several conventions are used throughout this guide. This is a brief overview of them.

  • Code examples and /paths/to/files are offset from text like so.
  • {item-zip} refers to the ZIP package that you purchased on ThemeForest.
  • {server} refers to the directory where Magento is installed on your server.
  • Definitions refer to functionality or Magento concepts with special meaning.

Notes contain important related info or tips.

Warnings contain critical advice or recommendations.

Install Magento

If you have Nexcess hosting, you can skip this. Magento is pre-installed and optimized by Nexcess. Lucky you!

If you must do things yourself, at least do yourself a favor: install Magento using SSH without sample data in your server root. Crucial has a decent guide about installing Magento via SSH.

$ wget http://www.magentocommerce.com/downloads/assets/1.9.0.1/magento-1.9.0.1.tar.gz
$ tar -zxvf magento-1.9.0.1.tar.gz
$ mv magento/* magento/.htaccess .
$ chmod o+w var var/.htaccess app/etc
$ chmod -R o+w media
$ chmod 550 mage
$ ./mage mage-setup .
$ rm -rf magento/ magento-1.9.0.1.tar.gz

Shell commands are marked with: $
If you don't understand what that means, get Nexcess hosting (notice a pattern?) and see: Hiring Us.

You must confirm these commands match the target Magento version indicated for this release. If a newer stable release is available from Magento, an older theme release will not work. With Magento you cannot mix/match core and theme releases. Modify these commands if necessary (it's your job to check).

Visit the URL where you installed Magento, and complete the setup. You'll need your DB info.

Magento Basics

Login to the Magento Admin

The default URL to access the Magento Admin is:
http://www.example.com/index.php/admin

After you've configured Magento to use URL rewrites on the frontend you can use:
http://www.example.com/admin

You can also setup a custom admin URL, but I leave that to you. Nexcess does this for improved security.

Magento Caches

Disable all caches in "Magento Admin > System > Cache Management".

Disabling caching makes changes to layouts/templates appear immediately, which is good for testing. It also puts more strain on the server because Magento has to fully process every request. You should re-enable caching before launching your website.

Magento Indices

Refresh all indices in "Magento Admin > System > Index Management".

Some admin operations will require different indices to be refreshed. Get familiar with this process.

Upload Files

Look inside the {item-zip}/httpdocs directory, which aligns to your Magento root directory. Ready your FTP application.

Local Code Files

Upload {item-zip}/httpdocs/app/code/local/ to {server}/app/code/local/ including all its files and subfolders.

If you visit the frontend now there may be an error because Magento's default/default package doesn't use Acumen's custom code. Ignore this error. After configuring Magento (in the next section) by setting the correct package/theme hierarchy the frontend will start working normally.

Local code overrides affect all stores powered by the Magento installation. This may lead to unexpected results and constrain a few custom features in Acumen.

Template + Layout Files

Upload {item-zip}/httpdocs/app/design/frontend/gravdept/ to {server}/app/design/frontend/gravdept/ including all its files and subfolders.

Skin Files

Upload {item-zip}/httpdocs/skin/frontend/gravdept/ to {server}/skin/frontend/gravdept/ including all its files and subfolders.

Categories

  1. Login to the Magento Admin
  2. Navigate to "Catalog > Manage Categories"

Notes on Categories

Look at the screenshot. You'll see a category named "Default Category" which is a Root Category. In Magento, every root category is assigned to a Store (and hence Store View) which you configured already.

Your first-level of navigation is created as sub-categories of the root category. You can drag/drop to change the nesting structure of categories. Most categories should have these options:

General Info > Name { your category name }
General Info > Is Active Yes
Display Settings > Is Anchor Yes (if you want to use layered navigation)

Uploading a category image will remove the title and description in the category header. The image should be 930px wide, but the height can be anything.

Notes on Navigation Menus

The primary navigation is optimized to show three-tier categories. This allows very complicated catalog shapes to be organized in a small area. If more than seven sub-categories exist, then sub-categories will be listed vertically to prevent running off the page and sub-sub-categories are hidden.

The primary navigation is generated programmatically (based on your catalog shape) and styled using CSS for consistency. For odd catalog shapes (very flat or narrow) the menus may be awkward. It is possible to designate specific layouts for specific categories, but this requires custom CSS and possibly hardcoding the menus depending on complexity.

Add the “Featured” Category

Create a Featured category to use later with the Category Slider. Write down the Category ID for later (see screenshot above for notes). Give this category special options to exclude it from the frontend navigation:

General Info > Is Active Yes
General Info > Include In Navigation Menu No
Display Settings > Is Anchor No

Widgets & Asset Context

Most UI Widgets have image assets for a specific context like the 'Home' page. Storing assets in a logical and consistent manner keeps you organized. All UI Widget assets follow this file pattern:

{server}skin/frontend/gravdept/acumen/media/{widget-name}/{context}/

In the Acumen Tour, all UI Widget examples have the tour context:

{server}skin/frontend/gravdept/acumen/media/promo-slider-right/tour/

Likewise on the 'Home' page, the promo-slider-right widget has the home context:

{server}skin/frontend/gravdept/acumen/media/promo-slider-right/home/

Each UI Widget may be used multiple in multiple places, and thus will have multiple contexts.

New Products Slider

A variable-width slider that shows products within the defined date range. Uses SlidesJS.

  1. Login to the Magento Admin
  2. Navigate to "Catalog > Manage Products"
  3. Edit a product
General > Set Product As New From Date Pick date
General > Set Product As New To Date Pick date

Server time may differ from local time. To mark a product as "new" immediately, set the "From Date" to yesterday. Then refresh the caches/indices.

Extensions

Theme and extension compatibility is an issue for all platforms, including Magento. Themes aren't tested with every extension for the same reason extensions aren't tested with every theme. It's a many:many relationship, so both sides try to coexist by working off the core Magento code.

Naturally, there can be conflicts when integrating custom code that was developed independently. Sometimes the theme or extension needs to be modified so they can work together.

In general, GravDept recommends avoiding extensions as much as possible. This minimizes your risk that future updates will break critical functionality. This is especially true for extensions that affect transactions and finances.

GravDept does not support or offer consulting for extension debugging.

Upgrades

Acumen is regularly updated with new features, improvements, fixes, and support for the latest Magento releases.

Magento's core development dictates how Acumen is developed. Magento does not maintain previous releases with patches or branches. New features, updates, and fixes always result in a new release. In this way, Magento avoids forward or backward compatibility conflicts because each new release deprecates the previous release. This is efficient and simple.

Acumen's development mirrors this practice. When Magento publishes a new release, then Acumen publishes a new release to maintain compatibility. The relationship between Acumen and Magento releases is many:one. For example:

  • Magento 1.5.0.1 had three Acumen releases: Acumen 1.2.5 Acumen 1.2.6 Acumen 1.2.7
  • Acumen 1.2.6 is for Magento 1.5.0.1 only.
  • Therefore Acumen 1.2.6 cannot be used with Magento 1.4.2.0 or Magento 1.6.0.0.

Acumen's development always results in a new release, which deprecates the previous release. New features, updates, and fixes are never retroactively added to previous releases. Acumen releases are always published for the latest stable Magento release.

Responsibility

  • You are responsible for upgrading Acumen and the Magento core.
  • You are responsible for merging your modifications.

Upgrading Magento

Magento core upgrades are tricky. Make full backups of the filesystem and database, then create a staging environment to test the core upgrade. If everything tests smoothly, then repeat the process for the production store — usually during off peak hours in case of problems. Be prepared to restore to your backups if necessary.

Never allow Magento's core to fall more than one release behind. Ignoring updates increases risk, and upgrading over multiple releases at once is more difficult.

Do not upgrade Magento's core until an official theme release supports it unless you plan to fully maintain the theme moving forward.

Upgrading Acumen

To upgrade Acumen, upgrading the Magento core will sometimes be required. Check the changelog for details.

The Easy Way

If you haven't modified Acumen, simply remove the previous release and follow the new release's documentation: {item-zip}/readme.html.

The Hard Way

If you have modified Acumen, merging your customizations will be required. File differencing applications like Kaleidoscope or DiffFork are useful in conjunction with the changelog.

Do not skip releases. The commit notes for each release assume all prior releases were completely applied.

Requesting Previous Releases

Only the latest Acumen release is bundled in the {item-zip}. If you require a previous release, send a support request indicating the exact version needed.
